# methodology.md — 先行指標の検証手法

本書は「ニュース化前の初動シグナル」を**推測でなくデータで**裏づけるための統計手続きを定義する。
実装(`src/validation/`, `src/scoring/`)はこの仕様に従うこと。

---

## 0. 基本原則

1. **推測でスコアを断定しない。** 取得不可の指標はスコアに算入せず「取得不可」と明示する。
2. **二重スコア。** `verified` のみの **Hardスコア** と、`proxy`/`estimated` を信頼度重み付きで足した **Extendedスコア** を併記。
3. **信頼度を定量化。** Extended には必ず confidence(%) と data coverage(verified割合) を付す。
4. **偽陽性を前提に設計する。** 多指標 × 多ラグの探索は必ず偽の相関を生む。再現性と多重比較補正で守る。

---

## 1. 特徴量の作成

各先行指標 `x_t` について以下を生成する（時点 t は「その時点までに入手可能」な値のみ使用）：

- 前年差 (YoY): `x_t - x_{t-365d}`
- 前月比 (MoM): `x_t / x_{t-30d} - 1`
- 移動平均: 窓 = 30, 90 日
- Zスコア: `(x_t - rolling_mean) / rolling_std`（窓は最低252営業日。短すぎる暗号オンチェーンは「信頼度低」フラグ）

対象資産価格 `p_t` について将来リターンを作成：

- `r_{t→t+h} = p_{t+h}/p_t - 1`、h ∈ {1, 7, 30, 60, 90, 120} 日

> **ルックアヘッド厳禁。** 特徴量は t 以前の情報のみ。将来リターンは t より後。両者の境界を必ずコードで保証する。

---

## 2. ラグ相関

指標特徴量 `f_t`(主に Zスコア) と将来リターン `r_{t→t+h}` の相関を、リードタイム
L ∈ {7, 14, 30, 60, 90, 120, 180} 日でずらして計算：

- `corr( f_{t-L}, r_{t→t+h} )`
- 指標が「価格に先行」していれば、ある L>0 で相関が立つ。
- ピアソンとスピアマン両方を出力（非線形・外れ値耐性のため後者を重視）。
- 出力: `outputs/lag_correlation_matrix.csv`（行=指標×特徴量, 列=L×h, 値=相関と n）。

### 多重検定の管理（必須）
- 1指標あたり |L|×|h| = 7×6 = 42 通り、指標数を掛けると数百〜千の検定になる。
- **Benjamini-Hochberg (FDR)** で p値を補正し、`q値` 列を追加。
- 「最良ラグ」を後付けで選ぶと過適合する → **リードタイムの採用は §4 の再現性条件を満たした場合のみ。**

### 見せかけ相関の排除（2026-06 追加・重要）
トレンドする系列（TVL・価格・供給量）のレベルZスコアは、強気相場では将来リターンと
**見せかけの相関**を生む（両者が同じトレンドで上昇するだけ）。実例: XRPL TVL のレベル相関は
0.857 だが、日次変化率相関はわずか 0.044。対策:

1. **定常化特徴量を見出しに使う**: ランク判定は変化ベースの特徴量
   （`_dz`=21日変化率のZスコア, `mom21`, `yoy252`）で行う。レベルZスコア(`_zscore`)は
   診断用に `spearman_r_level` として併記するのみ。
2. **トレンド共通交絡フラグ** `trend_confound`: `|レベル相関| ≥ 0.40` かつ `|変化相関| < 0.20`
   のとき True。該当指標はランクを **C 上限に降格**（見せかけを採用しない）。
3. **実効独立サンプルガード** `insufficient_history`: 重複する長期窓は実効サンプルを水増しする。
   `実効N ≈ 観測数 / (ラグ+ホライゾン)`。これが **10 未満なら C 上限に降格**。
   → 1-2年の履歴で 120-180日ラグを語ると実効Nは数個しかなく、相関値が高くても信頼不可。

## 3. 閾値イベント & イベントスタディ

各指標に閾値イベントを定義（**定常化Zスコア `_dz` > +1.5 を「初動」**。レベルZスコアだと
トレンド相場で「常にイベント」になり見せかけの的中率を生むため変化ベースを使う）：

1. イベント発生日 `t_e` を抽出。
2. **非重複窓に間引く（必須・2026-06 追加）**: 各ホライゾン h について、将来 h 日の窓が
   重ならないようイベント間隔を h 日以上に間引く。これを怠ると窓が重複し、ブートストラップの
   iid仮定が崩れて CI が過度に狭くなり有意性を過大評価する（実例: 17イベントの実効サンプルは
   約4-5）。`n_events`(非重複後) と `n_events_raw`(間引き前) を両方出力。
3. イベント後の将来リターン `r_{t_e→t_e+h}` を集計：
   - 平均 / 中央値リターン
   - 的中率 = `P(r > 0)`（方向性指標がマイナス効きなら符号反転）
   - 最大ドローダウン
   - シャープレシオ（イベント群リターンの平均/標準偏差、年率化注記）
4. **有意性検定**：
   - 1標本 t検定（H0: 平均リターン = 0）
   - **ブートストラップ 95%信頼区間**（リサンプル回数 ≥ 10,000、seed固定。少数イベント時は主指標）
5. 非重複イベント数 n が小さい(<10)場合は「信頼度低」を明示し、ランクは最大でも B 止まり。
- 出力: `outputs/event_study_results.csv`。

## 4. 有効性ランクの付与

単一指標・複合スコアの両方について付与する：

| ランク | 条件 |
|---|---|
| **A+** | スピアマン相関 ≥ 0.65 かつ 的中率 ≥ 70% かつ **複数のh/Lで再現** かつ FDR後も有意 |
| **A**  | 相関 ≥ 0.50 かつ 的中率 ≥ 65% かつ FDR後も有意 |
| **B**  | 相関 ≥ 0.35 かつ 的中率 ≥ 60% |
| **C**  | 上記未満だが符号が安定 → 補助指標 |
| **D**  | 不採用（ノイズ・サンプル不足・取得不可） |

### 再現性の定義（A+の肝）
- 隣接するリードタイム(例 L=60 と 90)で同符号・有意。
- **Walk-forward / out-of-sample**：期間を前半・後半に分け、両方で符号と有意性が一致するか。前半でしか出ない相関は採用しない。
- 出力: `outputs/indicator_scorecard.csv`（指標, レイヤー, data_quality, 最良L, 最良h, 相関, 的中率, q値, ランク, 採否, 信頼度コメント）。

> **業界指標 vs 個別指標の自動選択**：同一対象に対し業界指標と個別指標の両方が A 以上なら、
> サンプルが多くリードタイムが長い方（通常は業界指標）を優先採用する。これも scorecard に記録する。

## 5. スコアリング(`src/scoring/`)

### 二重スコア合成
- 各 Early Signal(採用された指標) を Zスコア化し 0–100 にマップ。
- **Hardスコア** = `verified` 指標のみを「(信頼度重み × ランク重み)」で加重平均。
- **Extendedスコア** = Hard に `proxy`/`estimated` を同様に加重して加える。
- **confidence(%)** = `Σ(採用指標の信頼度重み) / Σ(理論上満点の重み)` で算出し併記。
- **data coverage(%)** = `verified指標数 / その対象に紐づく全指標数`。

### XRP 専用スコア
- **総合実需スコア 0–100**：XRPL Tx, 成功Payment, AMM TVL, DEX出来高, Stablecoin TVL(verified)の加重合成。
- **XRPロック需要スコア 0–100**：因果連鎖
  `RLUSD供給↑ → XRPL利用↑ → AMM/ブリッジ/担保のXRPロック↑ → 流通量↓ → 価格↑`
  を、取得可能な代理変数で近似：
  - RLUSD供給(verified)
  - AMM内XRP残高(verified, ロックの直接代理)
  - XRPL Tx / 成功Payment(verified)
  - 担保/Lending/RWA(unavailable) → スコアに入れず confidence を下げる
  - 段階: 0–30 未発生 / 30–50 初動 / 50–70 加速 / 70–90 本格化 / 90+ 需給ショック
  > ロック需要は「担保利用」の直接データが無料で取れないため、**AMM内XRP残高を主代理**とし、
  > confidence は中程度で固定。本格化以上の判定には必ず「信頼度: 中／代理ベース」を明記する。

### 投資判断マッピング（強気/中立/弱気・買い時/売り時/撤退）
- スコア水準 + 変化速度(モメンタム) + confidence の3軸で判定。
- confidence が低いシグナルは判定を1段保守化（例: 「強気」→「中立(要確認)」）。
- 出力: `outputs/portfolio_signal_scores.csv`, `outputs/daily_report.md`。

## 6. 再現性・データ衛生

- すべての取得は `data/raw/` にスナップショット保存（取得日時付き）。
- ログに **データソース・取得日・行数・欠損率** を必ず記録。
- 欠損処理：前方補完は最大 N 日まで等、ルールを明示しコメント。補完した点はフラグを残す。
- 乱数(ブートストラップ)は seed 固定。
- テスト：特徴量(Zスコア/YoY)とラグ相関・イベントスタディの数値ロジックを `tests/` で検証。

---

## 7. 既知の限界（正直な明示）

- 暗号オンチェーンの履歴は短く、AMM等は新しい → 統計的検出力が低い。「信頼度低」を常に付す。
- 四半期ファンダ(NVIDIA売上, CAPEX, TSMC capex)は遅延が大きく、真の「先行」性は限定的。
- HBM価格・CoWoS稼働率・BBレシオ生値・取引所残高・SpaceX評価額は無料では取得不可 → スコア非算入。
- proxy(株価バスケット)は「指標が価格に効く」のではなく価格同士の相関になりうる。
  → proxy由来シグナルは Extended のみ、かつ confidence を割り引く。
